# Audit SEO — karibteck.com

**Date** : 2026-06-10
**Méthode** : skill `coreyhaines31/marketingskills@seo-audit`, appliqué à l'inspection du `dist/` réel (36 pages générées) sur la branche `feat/astro-refonte`.
**Contexte** : agence web locale, site statique Astro bilingue FR/EN. Objectif SEO = **local DOM (Guadeloupe, Martinique, Guyane…) + GEO (moteurs IA)**.

> Le JSON-LD est rendu en HTML statique (pas injecté par JS), donc directement inspectable — la limite habituelle de détection de schema ne s'applique pas ici.

---

## Résumé exécutif

**Santé globale : solide (B+).** Les fondations techniques sont excellentes : site statique zéro-JS (Core Web Vitals quasi-parfaits par construction), JSON-LD `Organization`/`ProfessionalService` sur **les 36 pages** (validé en CI), hreflang FR/EN + `x-default` partout, canonicals auto-référents cohérents (trailing slash), **un seul `<h1>` par page**, sitemap + `robots.txt` + couche GEO (`llms.txt`/`ai.txt`).

Les faiblesses relèvent du **contenu et du réglage fin**, pas de fondations cassées.

**Top 5 priorités :**

1. Pages `noindex` présentes dans le sitemap
2. NAP / schema local incomplet
3. Pas d'image OpenGraph
4. Meta descriptions / titres hors longueur
5. Contenu mince (blog 2 articles, pages tags/services courtes)

---

## Findings techniques

### 1. Pages `noindex` présentes dans le sitemap

- **Impact** : Moyen · **Priorité** : Haute (quick win)
- **Evidence** : `dist/sitemap-0.xml` contient `/merci/` et `/en/thank-you/`, qui portent `<meta name="robots" content="noindex">`. Signal contradictoire.
- **Fix** : filtrer ces routes dans `@astrojs/sitemap` (`filter: (page) => !/\/(merci|thank-you)\//.test(page)`).

### 2. NAP / schema local incomplet

- **Impact** : Haut (objectif = SEO local DOM) · **Priorité** : Haute
- **Evidence** : le nœud `Organization` n'expose que `addressRegion` + `addressCountry` (pas de `streetAddress`/`postalCode`/`addressLocality`), `telephone` placeholder (`+590 690 00 00 00`), **`openingHours` absent** (alors que les horaires existent en texte dans la section contact), **`sameAs` absent** (réseaux sociaux en `#`).
- **Fix** : compléter l'adresse réelle ; câbler `openingHours` depuis `CONTACT_DETAILS.hours` ; ajouter `sameAs` dès que les URLs sociales réelles existent. _(Dépend en partie des vraies données.)_

### 3. Aucune image OpenGraph

- **Impact** : Moyen · **Priorité** : Moyenne
- **Evidence** : `og:image` absent sur **les 36 pages** (génération Satori volontairement différée en Phase 3).
- **Fix** : générer une image OG (Satori) ou au minimum une image statique par défaut → meilleur rendu sur réseaux sociaux et certains aperçus IA.

## Findings on-page

### 4. Meta descriptions hors plage 150-160

- **Impact** : Moyen · **Priorité** : Haute (quick win)
- **Evidence** :
  - Pages services **tronquées (>160)** car elles réutilisent `service.intro` : `services/creation-sites-web` **197**, `intelligence-artificielle` 175, `hebergement-devops` 173, `logiciels-metier` 171, `applications-mobiles` 165.
  - Pages **tags quasi-vides (13-22 car.)** : `Articles : seo` = **14 caractères**.
  - Légales un peu courtes (~105-118).
- **Fix** : champ `metaDescription` dédié (≈155 car.) pour les services ; description générée riche pour les pages tags (« Nos articles sur X : conseils… »).

### 5. Titres trop longs (tronqués en SERP, >60)

- **Impact** : Faible-Moyen · **Priorité** : Moyenne
- **Evidence** : `en/blog/...fast-website...` **91**, articles blog FR **81-85**, `en/services/creation-sites-web` **71**. À l'inverse tags/légales courts (25-35) — acceptable mais peu optimisé.
- **Fix** : raccourcir les titres d'articles longs ; suffixe de marque plus court.

---

## Findings contenu

### 6. Contenu mince

- **Impact** : Moyen (concurrentiel en local) · **Priorité** : Moyenne
- **Evidence** : blog = **2 articles** ; **pages tags à 1 article** chacune (catégories fines) ; pages services = ~1 paragraphe + 4 puces.
- **Fix** : publier régulièrement ; étoffer chaque page service (process, cas d'usage local, FAQ par service) ; ne pas générer de page tag tant qu'elle n'a pas ≥3 articles.

### 7. Marqueurs d'écriture IA

- **Impact** : Faible · **Priorité** : Faible
- **Evidence** : **14 tirets cadratins (—)** rien que sur l'accueil (motif signalé par la référence `ai-writing-detection.md` du skill).
- **Fix** : passe éditoriale humaine (réduire les `—`, varier les formulations) avant lancement.
